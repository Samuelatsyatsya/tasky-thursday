YANGI_PRINCIPLE

It simple but effect and it accroynm is "You ain't gonna need it" which invented as part of programming or xp for short
It is a development principle in software development that emphasizes not adding functionality or features to a system until when, why and how they will be effective to the working system.

Benefits of YANGI_PRINCIPLE.

1. Improved Risk Management
2.Faster Development
3. Reduced Complexity
4. Simplicity and Maintainability LIKE (kISS)

And many others you can think of

SECTORS WHERE YANI_PRINCIPLE CAN BE USED:

1. Documentation
2. Code Refactoring
3.Resource Allocation on certain task
4. Designing
